## 1. The one-paragraph product context

Water is a Tauri 2 desktop writing app with one heretical UX rule: **the writer never types into a chat box and the LLM never sees a chat box.** The LLM sees only the manuscript text plus the click history of small "pills" that surface in the margin while the writer works. Pills come from named "speakers" — either generic personas (Echo / Architect / Editor / Cartographer / Chorus, all shipping in M2) or, in M3, character voices drawn from LSM v2.1 sheets — and their language is reactive/observational, never instructive. The writer can expand a pill into a bouquet of three child pills, expand any child into another bouquet, regenerate a bouquet, pin a pill to a quiet column at the right margin, or dismiss the whole thread. Local-first with cloud opt-in.

**M2 shipped the engine.** M3 wires character voices into it.

## 3. The M3 spec, in your hands

The locked v1 design spec is `docs/superpowers/specs/2026-05-16-water-design.md`. **Read § 4.5 (M3 — Character Sheets)** and **§ 3.5 (Character schema)** and **§ 6.2 (voice router, character branch)** before brainstorming.

**M3 ships:**

1. **Conversational Intake** — reusable component. Takes a JSON-Schema-like descriptor of questions (label, helper, type, validation, optional-skip). Renders one question at a time in a soft glowing popup, walks the schema, writes the result. Animations: gentle fade + subtle vertical glide between questions. Skippable per question. Resumable mid-flow.
2. **LSM v2.1 schema descriptors** — four schemas: main, bonus_traits, arc, perspectives. Each field carries a `prompt_question`, optional `helper`, optional `examples`.
3. **Character sheet view** — fully editable document view of a character. Inline editing, autosave. "Continue intake" button if any sections remain incomplete.
4. **Character index** — soft grid of glowing character cards (avatar placeholder, role hue, want/need teaser). Sortable, searchable.
5. **Scene ↔ character linking** — multi-select in scene metadata + auto-suggest based on names appearing in scene text.
6. **Character voice prompt template** — `prompts/speakers/character/template.toml` rendered per character via the LSM sheet. Goes through the M2 prompt assembler.
7. **Wire `character_dissonance` and `idle_pause_with_present_character`** — fill in the M2 trigger stubs.
8. **CharacterRegistry** — populate `CharacterRegistry::list()` from the `character` SQLite table.

**Exit criteria** (verbatim from spec § 4.5):

- Create a new character → walk full intake in ≤ 3 minutes → result matches LSM v2.1 PDF schema.
- Edit any field; on save, the corresponding `.toml` in `characters/` reflects the change.
- Open the character `.toml` in any text editor — it is human-readable.
- Character voice fires in an appropriate trigger; voice routing prefers POV character when present.

**Explicitly NOT in M3:**

- World Bible (M4)
- Heatmap audiovisualizer (M5)
- Macro spatial scene canvas (M6)
- Customization, polish, tester build (M7)

## 4. The absolutely-do-not-violate constraints

Re-emphasizing the M2 hard constraints — these still apply:

1. **The LLM NEVER receives conversational input from the user.** Even during Conversational Intake — the writer's intake answers ARE the manuscript-adjacent context. There is no chat textbox.
2. **Reactive / observational pill tone.** Character voices in M3 must obey the same tone rules. `you should / consider / try / I think you / as an AI` are blacklisted in the same three layers (prompt clause, PASS, post-hoc regex).
3. **Universe-first, personas-second.** With M3, this constraint LIVES: pills should prefer character voices when a character has standing. The voice router's `preferred_track: Character` path becomes real.
4. **Local-first.** Same as M2.
5. **Determinism on the orchestrator side.** `(trigger, scene_state, project_state, persona_state, character_state)` produces the same prompt assembly + speaker selection.
6. **Human-readable on disk.** Character `.toml` files in `characters/` are the truth; SQLite is a rebuildable index. Same contract as scenes.
7. **Plugin-contract design.** `CharacterSpeaker` is a `Speaker` impl alongside `PersonaSpeaker`. v2 loader can add `PluginSpeaker` against the same trait.
8. **Pastel-glow visual identity.** Conversational Intake popup uses the existing tokens. No new hex codes.
9. **Notion/Apple-fluid motion.** Intake transitions are gentle fade + vertical glide (~240ms).

## 7. Hard-earned lessons from M2 (carry these forward)

### Process

- **The combined phase-review pattern works.** Phase B got a combined spec+code review at end of phase; identified two important bugs (recent_word_delta semantics, missing list-item block IDs) that were fixed in one follow-up commit. Use this for M3 phase boundaries.
- **Implementers should report `DONE_WITH_CONCERNS` when ANY adaptation from the plan was needed.** The pattern that emerged: plan code is often clippy-flagged or assumes API names the codebase doesn't have. Adaptations are universally necessary and acceptable; document them in the report.
- **Use `task_id` to resume a subagent only when the work is half-done.** Fresh dispatches with full task text outperform context-resumed ones.
- **`cargo fmt --check` was missed twice in M2 (post-T3, post-T26).** Add it explicitly to every Rust task's gate list AND verify it runs.

### Vitest / TypeScript gotchas (M2-specific)

- **The cancellation-race fix is critical for every `useEffect` + `await onWaterEvent`.** T4 follow-up introduced the pattern; T19-T23 + T28 copied it. The shape:
  ```tsx
  useEffect(() => {
    if (!enabled) return;
    let cancelled = false;
    let unsub: (() => void) | undefined;
    (async () => {
      const u = await onWaterEvent("...", handler);
      if (cancelled) { u(); return; }
      unsub = u;
    })();
    return () => { cancelled = true; unsub?.(); };
  }, [enabled]);
  ```
- **`noUncheckedIndexedAccess: true` is on.** Regex match results need `?.[1] ?? ""`. Array indexing in tests needs `!` or explicit guards.
- **`pnpm --filter @water/app test src/path/file.test.tsx` works; `-- --run` does not.** pnpm 9 swallows `--run`.
- **Mock the editor at the canvas boundary, not from within.** EditorCanvas.test.tsx + PillLayer.test.tsx + PinnedColumn.test.tsx all use `vi.mock("../<sub>", ...)` to isolate. Tests own their concerns; component tests own subcomponent behavior.
- **`vi.hoisted` is the right pattern for top-level mocks.** Plain const mock refs get hoisted out of scope.

### Rust gotchas (M2-specific)

- **`#[allow(clippy::match_same_arms)]` is necessary on state-machine transitions** that produce the same output from semantically different states (e.g., `Dismissed` from `LlmFailed` vs. `UserDismiss`). Merging them obscures the state table.
- **Use `is_none_or` over `map_or(true, ...)`** (Rust 1.82+). `unnecessary_map_or` is on by default.
- **Cast via `u128::from(x)`, never `x as u128`.** `cast_possible_truncation` is enforced.
- **Plan TOML examples may need adaptation for `field_reassign_with_default`.** Use struct-update syntax (`AnalysisSnapshot { flow: Some(0.3), ..Default::default() }`) instead of mutable init.
- **`HashSet<T, S>` parameters need `S: BuildHasher` generic** to avoid `implicit_hasher`.

### Windows / PowerShell (carries from M2 handoff)

- **PATH setup at every fresh shell.** `$env:Path = "$env:USERPROFILE\.cargo\bin;$env:Path"`.
- **`Set-Content -Encoding UTF8` writes a BOM** in PowerShell 5.1. Use `[System.IO.File]::WriteAllText(..., (New-Object System.Text.UTF8Encoding $false))` when generating JSON fixtures (T29 ran into this).
- **CRLF/LF git warnings** are environmental. Ignore.
- **`NativeCommandError` on cargo stderr** is cosmetic. Ignore.
