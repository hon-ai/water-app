## 1. The one-paragraph product context

Water is a Tauri 2 desktop writing app with one heretical UX rule: **the writer never types into a chat box and the LLM never sees a chat box.** The LLM sees only the manuscript text plus the click history of small "pills" that surface in the margin while the writer works. Pills come from named "speakers" — either generic personas (Echo / Architect / Editor / Cartographer / Chorus) or, later, character voices drawn from LSM v2.1 sheets — and their language is reactive/observational, never instructive. The writer can expand a pill into a bouquet of three child pills, expand any child into another bouquet, regenerate a bouquet, pin a pill to a quiet column at the right margin, or dismiss the whole thread. Local-first with cloud opt-in; bundled tiny quantized local model is the fallback so the app works offline on day one.

M2 builds that engine.

### Key constants and patterns to memorize

- **`Db: Send + !Sync`** because `rusqlite::Connection` contains a `RefCell`. Shared as `Arc<Mutex<Db>>` in `OpenProject`. M2 code that touches the DB must follow the lock-then-clone-then-drop pattern from `app/src-tauri/src/commands/scene.rs::scene_rename`.
- **`#![forbid(unsafe_code)]`** at `crates/water-core/src/lib.rs:6`. Can't be locally overridden — affects test design (no `env::set_var`).
- **`#![allow(clippy::missing_errors_doc)]`** at `lib.rs:8`. No `# Errors` doc needed on pub fns returning `Result` (but write them anyway when they help readers).
- **`rusqlite::Statement` is `!Send`.** Async Tauri command handlers can't hold one across `.await`. Scope queries in their own block returning owned `Vec`.
- **Tauri 2 capabilities:** `core:default` + `dialog:default` in `app/src-tauri/capabilities/default.json`. M2 may need more (e.g., `clipboard:default` if pill text is copyable, `notification:default` if we surface "tone violation" warnings).
- **`Id` = ULID newtype** via `crate::Id`. `as_str()`, `parse()`, `Clone`, `FromStr`.
- **Block IDs** = `^bk-XXXX` Obsidian-compatible tokens, ensured by `crate::block::ensure_block_ids`. Duplicates are tolerated (KNOWN_FRAGILE #2).

## 3. The M2 spec, in your hands

The locked v1 design spec is `docs/superpowers/specs/2026-05-16-water-design.md`. **Read § 4.4 (M2 — Editor & Pill Engine)** and **§ 6 (Pill orchestrator state machine)** and **§ 6.3 (tone enforcement)** before brainstorming. The relevant scope:

**M2 ships:**
1. **Editor bake-off** — build identical small prototypes in ProseMirror and Lexical. Decision criteria:
   - block-ID maintenance ergonomics
   - decoration API for pill highlights + pill-snippet underlines
   - selection/mark stability under autosave write-backs
   - bundle size impact
   - perf on a 50 000-word scene
   - long-undo behavior
   Winner committed before rest of M2 proceeds.
2. **Block editor** — paragraph, scene break, dialogue, heading-2/3, ordered/unordered list. Block IDs maintained automatically.
3. **Typing telemetry** — idle detector, sentence-end / paragraph-end / mid-sentence cursor classifier, structural-inflection detector. Emitted as events to Rust core.
4. **Pill orchestrator** (Rust) — deterministic state machine. Pure, unit-tested.
5. **Voice router** (Rust) — chooses persona-track speaker (character voices are M3). Deterministic for `(trigger, scene_state, project_state)`.
6. **Prompt library** (Rust) — TOML templates per `(speaker_kind × trigger × task)`. Hot-reloadable in dev. Tone enforced globally.
7. **Bouquet generator** — exactly N=3 variants in structured JSON. Regenerate adds prior-variant-exclusion clauses.
8. **Anti-loop** — temperature bump + diverge-clause when consecutive bouquets at the same node have ≥ 70 % token overlap.
9. **Pill UI** — pastel-glow capsule in margin. Hover dims rest of page + glow line to snippet. Click expands to bouquet of 3 stacked sub-pills + translucent regenerate + pin + X. Pinned pills move to right-edge quiet column at half-opacity.
10. **Rabbit hole** — unlimited depth. Prior siblings collapse to thin glow lines on the left edge. Breadcrumb chain at top. Only current bouquet fully rendered. X closes thread.
11. **Persona registry** — 5 default named personas, renameable per project, each with distinct hue and tone profile:
    - **Echo** — muse; emerges what's almost-there
    - **Architect** — structure / pace
    - **Editor** — diction / clarity
    - **Cartographer** — world / setting consistency
    - **Chorus** — eliciting mode only

**Exit criteria** (verbatim from spec):
- Idle 3 s after a paragraph → at most one pill surfaces within 1.5 s of the analysis completing.
- Two pills max on screen; mid-sentence typing never surfaces a pill.
- Expanding a pill shows exactly 3 sub-pills; regenerate produces 3 different ones.
- Rabbit hole works at arbitrary depth; breadcrumb collapse visible; anti-loop fires when configured.
- Pinning a pill persists it across app restart; dismissed pills do not.
- **Tone audit: 0 instances of "you should", "consider", "try", "I think you", "as an AI" in 200 sampled pills.**

**Explicitly NOT in M2:**
- Character voices (deferred to M3 — they ride the same engine M2 builds)
- Conversational Intake popup (M3)
- LSM v2.1 character sheets (M3)
- World Bible (M4)
- Heatmap audiovisualizer (M5)
- Macro spatial scene canvas (M6)
- Customization, polish, tester build (M7)

## 4. The absolutely-do-not-violate constraints

These are non-negotiable. They define the product. A reviewer will reject any PR that breaches them.

1. **The LLM NEVER receives conversational input from the user.** Only manuscript text + pill click history. There is no chat textbox in this app, not even a hidden one, not even in dev tools. If you find yourself reaching for one, stop.
2. **Reactive / observational pill tone.** Words like "you should", "consider", "try", "I think you", "as an AI" are blacklisted in the tone audit. Pills observe, name, mirror, ask — they do not advise. Read § 6.3 of the design spec for the full tone grammar.
3. **Universe-first, personas-second.** Character voices (M3) outrank personas. Pills speak as the universe's residents whenever possible; personas are the fallback when no character has standing to speak to a particular trigger. Personas fade visually as a project's character roster matures.
4. **Local-first.** The default LLM provider is "best available," but the bundled tiny quantized local model must always be able to surface pills offline. Cloud is opt-in per project.
5. **Determinism on the orchestrator side.** Given a fixed `(trigger, scene_state, project_state, persona_state)`, the voice router and prompt assembly produce the same output. Variation lives only in the LLM's sampling. This makes pill behavior testable.
6. **Human-readable on disk.** Markdown + TOML are the truth. `project.db` is a rebuildable index. M2 must not introduce binary on-disk artifacts that lose information if the DB is deleted. (Snapshots are zstd — that's fine because they're optional history, not truth.)
7. **Plugin-contract design now, plugin loader in v2.** When you design the pill orchestrator, the voice router, and the prompt library, design them with "third-party plugins will inject triggers, personas, prompts, and post-filters here" in mind. Don't ship the loader; do ship the seams.
8. **Pastel-glow visual identity, no hard corners, glow-only elevation.** Pill UI must obey the existing tokens in `app/src/styles/tokens.css`. No shadows under 6 px blur, no 1 px borders, no opaque overlays. Hover dims the rest of the page; nothing else.
9. **Notion/Apple-fluid motion language.** Pill emergence is a soft fade-in over ~280 ms. Bouquet expansion is a vertical glide of ~240 ms ease-out-soft. No bounce. No spring. Reduced-motion mode collapses everything to 0 ms.

### Step 1: Brainstorm session (REQUIRED, use the skill)

- **Local model fallback** — when network is down and no local model is bundled, what does the pill UI show? (Nothing? A muted "thinking…" capsule? The orchestrator stays silent?)
- **Editor library choice** — set up the ProseMirror-vs-Lexical bake-off as the first concrete sub-task.
- **Tone enforcement layer** — runtime regex blacklist? Post-LLM filter that rejects+resamples? Pre-prompt instruction? Probably all three.

## 7. Hard-earned lessons from M1 → M1.5.1 (carry these forward)

These cost real time to discover. Inheriting them saves the next agent the same discovery cycle.

### Process

- **Plan amendments belong in their own `plan(TN):` commit** after the implementation commit. Update both the amendment block at the bottom of the plan AND any task descriptions whose scope changed. M1.5 had Amendment 1 (T7→T12 deletion deferral); M1 had nine amendments. They are normal and good when an implementer correctly diagnoses a real plan defect.
- **Pre-authorize lint fixes by name** in implementer prompts. The recurring offenders: `must_use_candidate`, `doc_markdown`, `single_match_else`, `implicit_hasher`, `inherent_to_string`, `cast_precision_loss`, `cast_possible_truncation`, `uninlined_format_args`, `duration_suboptimal_units`, `assigning_clones`. Saves a fix-round per task.
- **Every commit must be green.** If a plan step would break tests until a later step lands, the plan is wrong — amend it. Amendment 1 in M1.5 caught exactly this: T7 wanted to delete PlaceholderEditor mid-milestone but five files transitively depended on it. Deletion was deferred to T12 (the cleanup task).
- **Combined spec+quality review in one subagent dispatch** is appropriate for clean tasks. Separate reviews only when fix loops happen.
- **Final milestone review is a hard end-gate.** Dispatch `superpowers:requesting-code-review` against the full `m1.5..m2` range before tagging. The M1.5 review surfaced four real fixes worth landing as a `.1` patch.

### Vitest gotchas (these WILL bite the next agent if not pre-authorized)

- **`vi.mock` hoisting + top-level mock identity:** the literal pattern
  ```ts
  const xMock = vi.fn();
  vi.mock("y", () => ({ x: xMock }));
  ```
  throws `ReferenceError: Cannot access 'xMock' before initialization` in Vitest 2.1.x because `vi.mock` is hoisted above the const. Always use `vi.hoisted`:
  ```ts
  const { xMock } = vi.hoisted(() => ({ xMock: vi.fn() }));
  vi.mock("y", () => ({ x: xMock }));
  ```
- **`vi.useFakeTimers({ shouldAdvanceTime: true })` is required when combining fake timers with `waitFor`.** Without `shouldAdvanceTime`, RTL's microtask scheduler stalls. Set in `beforeEach`.
- **`pnpm --filter @water/app test -- --run <path>` does NOT work in pnpm 9.** pnpm intercepts `--run` as its own flag. The `test` npm script is already `vitest run` (non-watch), so plain positional path works: `pnpm --filter @water/app test src/path/to/file.test.tsx`.
- **`HTMLDialogElement.prototype.showModal` and `.close` are missing in jsdom.** Stub them in `beforeEach` (see `app/src/sheets/CreateProjectSheet.test.tsx`). M2's pill UI may or may not use `<dialog>` — if you avoid it you avoid this. If you use it, copy the stub.
- **`<aside aria-label="X">` has implicit role `complementary`.** Useful for testing without `data-testid` clutter.

### Rust gotchas

- **`rusqlite::Statement: !Send`** — async Tauri handlers can't hold one across `.await`. Scope queries in their own block returning owned `Vec`.
- **`Arc<Mutex<Db>>` discipline:** the canonical pattern in `app/src-tauri/src/commands/scene.rs::scene_rename`:
  ```rust
  let (db, root) = {
      let proj = state.project.lock().await;
      let project = proj.as_ref().ok_or("no project open")?;
      (project.db.clone(), project.root.clone())
  };
  // ... drop state.project lock implicitly here ...
  let db_guard = db.lock().await;
  let store = SceneStore::new(&db_guard, root);
  // ... use store ...
  ```
  Never hold the state lock during DB work.
- **`#![forbid(unsafe_code)]`** at the crate root means tests can't use `env::set_var` or any other unsafe-requiring escape hatch. Use direct struct construction in tests.

### Windows / PowerShell

- **Every fresh PowerShell session needs:**
  ```powershell
  $env:Path = "$env:USERPROFILE\.cargo\bin;$env:Path"
  ```
  before any `cargo` call. Put this at the top of every implementer prompt.
- **`NativeCommandError` on cargo stderr** is cosmetic — cargo writes informational lines to stderr and PowerShell flags them. Ignore.
- **CRLF/LF git warnings** are environmental. Ignore.
- **The `bash` tool aborts silently** on `write` content over ~1500 lines. Workaround: write the file header, then append the rest via `edit` with unique anchor strings.
- **Unicode arrows (`→`)** can mangle when PowerShell pipes a here-string into `git commit -m`. Implementers have used `->` as fallback. Either works; consistency matters within a single commit message.
