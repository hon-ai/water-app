# M5 Heatmap Audiovisualizer — Implementation Plan

**Status:** Open. Built on `m4` (2026-05-20) + the M4.5 polish sprint (HEAD at `648af45` light-mode refresh, `b4c381d` spec). Targets the `m5` tag when all tasks ship green.

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the per-scene Heatmap strip — a 24 px-tall ambient barometer above the editor title — visualizing five metrics (Pacing, Valence, Coherence, Presence, World-refs) with paragraph-precision drag-to-scroll. Pacing on by default, others toggle in via a popover; Valence + Coherence cache per text-hash so steady-state recompute is ~5 LLM calls per autosave; a per-session token budget prevents runaway cost. Migrate `pace_floor` to read its signal from the heat cache. Tag `m5` when done.

**Architecture:** ~19 tasks across 7 phases. **Phase A** (engine foundations, 5 tasks) lays down the v5 migration, the `heat` module, `HeatStore`, and the three local-only metric computers (pacing/presence/world-refs). **Phase B** (LLM-backed metrics, 3 tasks) implements valence, coherence, and the per-session token budget with graceful degradation. **Phase C** (trigger migration, 2 tasks) rewires `pace_floor` and `scene_flow_dip` to read from `HeatStore`. **Phase D** (IPC, 2 tasks) adds the Tauri commands + TS types. **Phase E** (UI, 5 tasks) builds `HeatmapStrip`, the metric picker popover, the phrase-bank-driven tooltip, the drag-scrub handler, and the one-time intro overlay. **Phase F** (audit, 2 tasks) builds the eval fixture, walks the manual smoke, tags `m5`, and writes the M6 handoff.

**Tech Stack:** Rust 1.85 + tokio + rusqlite + serde + reqwest (existing); React 18 + TypeScript strict (existing); Tauri 2 (existing). No new third-party deps.

**Spec:** `docs/superpowers/specs/2026-05-20-m5-heatmap.md`

## Phase A — Engine foundations

Lays down the v5 migration, the heat module, store, and the three local-only metrics. All Rust, no Tauri or UI yet. Every task ends with a green `cargo test -p water-core` and a commit.

### Task 1: v5 SQLite migration

**Files:**
- Create: `crates/water-core/sql/v5_heatmap.sql`
- Modify: `crates/water-core/src/migrations.rs`

**Shape:**
```sql
CREATE TABLE heat_metric (
    scene_id      TEXT NOT NULL,
    paragraph_ix  INTEGER NOT NULL,
    metric        TEXT NOT NULL,
    value         REAL NOT NULL,
    text_hash     TEXT NOT NULL,
    updated_at    TEXT NOT NULL,
    PRIMARY KEY (scene_id, paragraph_ix, metric),
    FOREIGN KEY (scene_id) REFERENCES scene(id) ON DELETE CASCADE
);
CREATE INDEX heat_metric_by_scene ON heat_metric(scene_id, metric);

CREATE TABLE scene_typing_history (
    scene_id    TEXT NOT NULL,
    ts_ms       INTEGER NOT NULL,
    word_delta  INTEGER NOT NULL,
    PRIMARY KEY (scene_id, ts_ms),
    FOREIGN KEY (scene_id) REFERENCES scene(id) ON DELETE CASCADE
);

UPDATE schema_version SET version = 5;
```

Tests: PRAGMA-based column + index existence assertions, mirror the M4 T1 pattern.

### Task 2: `heat` module skeleton

**Files:**
- Create: `crates/water-core/src/heat/mod.rs` (re-exports + `HeatMetricKind`, `HeatRow`)
- Create: `crates/water-core/src/heat/types.rs` (the type definitions)

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum HeatMetricKind {
    Pacing,
    Valence,
    Coherence,
    Presence,
    WorldRefs,
}

impl HeatMetricKind {
    pub fn as_str(self) -> &'static str { /* "pacing" etc. */ }
    pub fn requires_llm(self) -> bool { matches!(self, Self::Valence | Self::Coherence) }
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct HeatRow {
    pub paragraph_ix: u32,
    pub value: f32,
    pub text_hash: String,
    pub updated_at: String,
}
```

### Task 3: `HeatStore` CRUD

**Files:**
- Create: `crates/water-core/src/heat/store.rs`

```rust
pub struct HeatStore<'a> { db: &'a Db }

impl HeatStore<'_> {
    pub fn read(&self, scene_id: &Id, kind: HeatMetricKind) -> Result<Vec<HeatRow>>;
    pub fn write_batch(&self, scene_id: &Id, kind: HeatMetricKind, rows: &[(u32, f32, &str)]) -> Result<()>;
    pub fn invalidate(&self, scene_id: &Id, paragraph_ix: u32) -> Result<()>;  // deletes all metrics for that ix
    pub fn invalidate_all(&self, scene_id: &Id) -> Result<()>;
}
```

`write_batch` uses one transaction. `invalidate` per-paragraph drops the rows so the next compute pass refills them; mirrors the M4 store-write-then-fs pattern.

Tests: insert + read round-trip, replace-same-key updates value, invalidate drops only the targeted paragraph, FOREIGN KEY cascade on scene delete.

### Task 4: Local metrics — pacing

**Files:**
- Create: `crates/water-core/src/heat/compute.rs` (start with pacing)
- Create: `crates/water-core/src/heat/paragraph.rs` (`paragraph_offsets` helper)

`paragraph_offsets(body: &str) -> Vec<usize>` finds the byte-offset of each paragraph break (double-newline). Used by every metric to align scores to paragraphs.

`compute_pacing` reads `scene_typing_history` for the scene + the current paragraph offsets, computes a rolling-window word-delta per paragraph (median of the trailing 250 ms windows that overlap each paragraph's typing span). Returns `Vec<(u32, f32)>` (paragraph_ix, score 0..=1).

Tests: hand-built history fixtures with deterministic timestamps.

### Task 5: Local metrics — presence + world-refs

`compute_presence(body, characters)` scans each paragraph for character name+alias matches (reuse M3 `character::autosuggest`). Returns counts normalized by scene-max.

`compute_world_refs(body, world_registry)` does the same for world entries (reuse M4 `WorldRegistry::find_by_token`). Returns counts normalized by scene-max.

Both functions are pure: scene snapshot in, `Vec<(u32, f32)>` out. The orchestrator service drives recompute and writes to `HeatStore` (Phase C/D).

Tests: fixtures with known counts.

**Phase A close-out.** ~26 new tests. v5 schema lands; local-only metrics work without an LLM.

## Phase B — LLM-backed metrics

Adds valence + coherence with per-text-hash caching + a session-level token budget.

### Task 6: `compute_valence`

**Files:**
- Create: `crates/water-core/src/heat/llm.rs`
- Create: `prompts/tasks/heat_valence.toml`

Prompt: "Rate the emotional valence of the following paragraph on a -1..=+1 scale. Reply with just the number." 1-token output, 200-token-ish input.

`compute_valence(router, paragraph)` → `Result<f32>`. Caches via the orchestrator's text_hash gate — only call when the cached row's text_hash differs from the current.

### Task 7: `compute_coherence`

Similar shape to valence. Prompt asks for a 0..=1 cosine-similarity-style score given the current paragraph + the previous 2 paragraphs as context.

(Local embedding models are M6+. For M5 we use the LLM as a coarse cosine approximator.)

### Task 8: Per-session token budget

**Files:**
- Create: `crates/water-core/src/heat/budget.rs`

`LlmBudget` tracks tokens consumed per session. Default cap: 20_000 input + 5_000 output. When exceeded, valence/coherence recompute returns `Err(BudgetExceeded)`; the orchestrator catches this and writes a sentinel value (NaN) to `HeatStore` so the renderer can show "budget reached" without spinning.

Tests: budget exhaustion + reset on project close.

**Phase B close-out.** ~14 new tests. All five metrics compute; LLM metrics gracefully cap.

## Phase C — Trigger migration

### Task 9: `pace_floor` reads from `HeatStore`

Current `pace_floor::evaluate` uses an ad-hoc word-count heuristic on the recent body window. Rewire to read the latest N pacing values from `HeatStore` and fire when the trailing average falls below a threshold.

Fallback: when `HeatStore` returns empty (M2 projects, first-paragraph case), keep the existing heuristic.

Existing `pace_floor` tests must continue to pass; add new tests asserting it reads `HeatStore` when populated.

### Task 10: `scene_flow_dip` reads from `HeatStore`

Same shape as Task 9 but reads the coherence metric.

## Phase E — UI

The visible part. Pacing-only by default; everything else opts in via the picker.

### Task 13: `HeatmapStrip` skeleton

**Files:**
- Create: `app/src/heat/HeatmapStrip.tsx`
- Modify: `app/src/chrome/EditorCanvas.tsx` (mount above title hairline)

Renders a 24 px-tall strip + a small "Pacing ▾" chip on the right edge. Per-paragraph cell width = canvas-width / paragraph-count. Cell opacity proportional to the pacing value. Fades in on mount via `water-pill-fade-in`.

### Task 14: `HeatmapMetricPicker`

**Files:**
- Create: `app/src/heat/HeatmapMetricPicker.tsx`

Popover triggered by clicking the metric chip. Five checkbox rows with one-line descriptions; toggling fires `ipc.heatSetMetricEnabled`. Each enabled metric draws a separate semi-transparent layer over the strip, hue-coded.

### Task 15: Hover tooltip + phrase bank

**Files:**
- Create: `prompts/heat/phrasebank.toml`
- Modify: `app/src/heat/HeatmapStrip.tsx` (hover handler)

`phrasebank.toml` has 8–12 phrases per (metric, intensity-bucket) — e.g., for Pacing × low: `"slow burn"`, `"holding"`, `"resting"`, `"unhurried"`. Renderer picks one stably per (paragraph, metric) via a deterministic hash so the tooltip phrase doesn't churn on every hover.

Tooltip floats below the strip, ~120 ms delay on enter, fades out fast.

### Task 16: Drag-scrub + click-to-scroll

**Files:**
- Modify: `app/src/heat/HeatmapStrip.tsx`

Pointer events: mousedown sets a `scrubbing` flag, mousemove maps x-position → paragraph index → `editor.scrollToParagraph(ix)`. On mouseup, the targeted paragraph flashes its hue-glow accent once (reuse the `water-pill-fade-in` pattern, applied to the block element).

Tests: drag from x=0 to x=width should sweep through paragraph 0 → last; click at fraction 0.5 jumps to the middle paragraph.

### Task 17: One-time intro overlay

**Files:**
- Create: `app/src/heat/HeatmapIntro.tsx`
- Modify: `app/src/heat/HeatmapStrip.tsx` (conditionally mount)

First time the writer opens any scene after M5 lands, a small tooltip appears under the strip: *"this is the heat. drag to find a spot."* with a dismiss-X. Persistence: `localStorage.setItem("water:heatmap-intro-seen", "true")`. Mirrors the scenes-collapsed key pattern.

**Phase E close-out.** Strip fully interactive; metric picker toggles work; intro tooltip surfaces once.

## Phase F — Audit, fixture, smoke, tag

### Task 18: `eval/m5_acceptance/` fixture

**Files:**
- Create: `eval/m5_acceptance/heat_demo.md` (1 scene, ~12 paragraphs, deliberately-shaped)
- Create: `eval/m5_acceptance/README.md` (expected strip pattern per metric)

Scene shape:
- Paragraphs 1–3: short clipped sentences (high pacing).
- Paragraphs 4–6: long dense paragraphs (low pacing).
- Paragraphs 7–9: emotionally cold (Valence < 0).
- Paragraphs 10–12: emotionally warm (Valence > 0).
- Mid-scene: a sudden topic jump (low coherence at the jump).
- Throughout: 2-3 character mentions (Presence track), 2-3 world entry mentions (World-refs).

README documents the expected strip pattern per metric so the manual smoke can compare visually.

### Task 19: Manual smoke + tag `m5` + write M6 handoff

Pre-tag: full `cargo test -p water-core` + `cargo test -p water-app` + `pnpm @water/app test` green.

Manual smoke against the fixture:
1. Open the fixture scene.
2. Pacing track should show high-density bars in 1–3 and 7–9, faint in 4–6 and 10–12.

4. Toggle Coherence → dip at the topic jump.
5. Toggle Presence → bars where characters appear.
6. Toggle World-refs → bars where world entries appear.
7. Drag scrubbing → editor scrolls and the targeted paragraph flashes.
8. Click the metric chip → popover opens; toggling persists across reload.
