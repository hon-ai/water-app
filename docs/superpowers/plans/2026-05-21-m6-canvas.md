# M6 Macro Spatial Canvas — Implementation Plan

**Status:** Open. Built on `m5` (2026-05-20). Targets the `m6` tag when all tasks ship green.

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (or execute task-by-task in the current session). Steps use checkbox (`- [ ]`) syntax.

**Goal:** Ship the macro spatial canvas — a fourth nav surface where the writer drags scene cards around a 2D pan/zoom canvas. Each card carries a tiny heat sparkline reusing the M5 strip's render logic. Position persists to SQLite + scene-md frontmatter. Reading-order overlay toggles between spatial layout and manuscript order. Auto-flow + groups + first-run intro overlay round out the v1.

**Architecture:** ~15 tasks across 5 phases. **Phase A** (engine, 3 tasks) — v6 migration + `SceneStore::set_canvas_position` + frontmatter round-trip + auto-flow helper. **Phase B** (IPC, 2 tasks) — Tauri commands + TS types. **Phase C** (UI refactor, 2 tasks) — extract `HeatmapStripView` presentational component from M5's strip; add the canvas nav target. **Phase D** (UI build, 6 tasks) — `CanvasSurface` with pan/zoom, `SceneCard` with sparkline, drag-to-move, reading-order overlay, metric chip, first-run intro. **Phase E** (audit, 2 tasks) — eval fixture + manual smoke + tag.

**Tech Stack:** Rust 1.85 + tokio + rusqlite + serde (existing); React 18 + TypeScript strict (existing); Tauri 2 (existing). No new third-party deps.

**Spec:** `docs/superpowers/specs/2026-05-21-m6-canvas.md`

**Parent design:** `docs/superpowers/specs/2026-05-16-water-design.md` § 4.8.

## Phase A — Engine

### Task 1: v6 SQLite migration

**Files:**
- Create: `crates/water-core/sql/v6_canvas.sql`
- Modify: `crates/water-core/src/migrations.rs`
- Test: `crates/water-core/src/migrations.rs` tests module

SQL:
```sql
ALTER TABLE scene ADD COLUMN canvas_x REAL;
ALTER TABLE scene ADD COLUMN canvas_y REAL;
ALTER TABLE scene ADD COLUMN canvas_group TEXT;
UPDATE schema_version SET version = 6;
```

Nullable on purpose. NULL = unplaced; renderer auto-flows on first paint.

Tests: column-presence PRAGMA assertions; schema_version = 6; pre-M6 scenes have NULL canvas fields after ratchet.

### Task 2: `SceneStore::set_canvas_position` + frontmatter round-trip

**Files:**
- Modify: `crates/water-core/src/scene.rs`
- Modify: `crates/water-core/src/scene_md.rs` (frontmatter)

API:
```rust
impl SceneStore<'_> {
    pub fn set_canvas_position(&self, scene_id: &Id, x: Option<f32>, y: Option<f32>) -> Result<()>;
    pub fn set_canvas_group(&self, scene_id: &Id, group: Option<&str>) -> Result<()>;
    pub fn reset_all_canvas_positions(&self, project_id: &Id) -> Result<()>;
}
```

Scene-md frontmatter gains optional `canvas_x` / `canvas_y` / `canvas_group`. On write, sync from DB. On rebuild (`rebuild::run`), pull from frontmatter into DB.

Tests: round-trip, NULL handling, reset clears every scene in a project (not just one).

### Task 3: `canvas::auto_flow`

**Files:**
- Create: `crates/water-core/src/canvas/mod.rs`
- Create: `crates/water-core/src/canvas/layout.rs`

```rust
/// Deterministic left-to-right rows by manuscript order. 8 cards per
/// row at canvas-space units of 240×140 (card 200×100 + 40×40 gap).
pub fn auto_flow(scenes: &[SceneRow]) -> Vec<(Id, f32, f32)>;
```

Tests: 0 scenes → empty; 1 scene → (0, 0); 9 scenes → second row; manuscript-order respected even when scene IDs sort differently.

## Phase D — UI build

### Task 8: `CanvasSurface` skeleton

**Files:**
- Create: `app/src/canvas/CanvasSurface.tsx`
- Create: `app/src/styles/canvas.css`

On mount: `Promise.all([ ipc.sceneCanvasList(), ipc.heatReadSettings() ])`. Maps each scene to a virtual position (use canvas_x/canvas_y if set; else auto-flow). Renders a positioned `<SceneCard />` per scene inside an `<svg>`-or-`<div>` container.

### Task 9: Pan + Cmd-zoom

Container is `position: relative; overflow: hidden`; child wrapper has `transform: translate(panX, panY) scale(zoom)`. Pointer-down on empty space starts pan; pointer-move updates translate. Cmd/Ctrl + wheel updates zoom around cursor.

Min zoom = fit-all-scenes; max zoom = 2×. Default zoom: fit-all if `> 3` scenes else 1.0.

### Task 10: `SceneCard`

**Files:**
- Create: `app/src/canvas/SceneCard.tsx`

Receives scene row + sparkline data + active metric. Renders title (title-serif), hairline, sparkline (using `HeatmapStripView`), footer (word count + paragraph count). Click → `App` state change to navigate to scenes nav with that scene active.

### Task 11: Drag-to-move

Pointer-down on a card starts drag; pointer-move updates virtual position; pointer-up commits via debounced `ipc.sceneCanvasSetPosition`. Drag also pans the underlying canvas if shift held? — for v1 no; keep drag scoped to the card.

### Task 12: Reading-order overlay

`<svg>` layer above cards, below interaction layer. Renders smoothed cubic-Bezier paths between scenes in manuscript order, ~30% opacity. Toggle via `O` keyboard shortcut. Off by default.

### Task 13: Metric chip + first-run intro

Reuse `HeatmapMetricPicker` to let the writer pick which metric the cards' sparklines render. Place the chip at top-right of the canvas surface (same chrome-corner pattern as the editor strip).

First-open intro: localStorage key `water:canvas-intro-seen`. Floating tooltip near the canvas center: *"drag scenes anywhere. cmd-scroll zooms. press O to toggle reading order."* with × dismiss.
