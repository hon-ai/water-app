## 1. One-paragraph product context

The writer sees pills surface in the margin; that's the *micro* feedback channel. The Heatmap is the *macro* feedback channel: a small, glanceable strip that visualizes the shape of a scene over time. Pacing, valence (warm vs. cold), coherence, presence (how thick the cast is), and world-references (how much of the world bible has been activated). When the writer is in flow they ignore it. When they pause, the strip becomes a way to see the scene's silhouette and find where to come back to.

It's not a chart. It's a barometer.

---

## 2. Design intent (the un-violable rules)

All M1–M4 hard constraints carry forward. M5 specifics:

1. **The Heatmap is ambient, not analytical.** It's not a dashboard with axis labels and numbers. The writer perceives the strip as a small horizon, not a graph. No tooltips that say "Pacing: 0.42." Maybe a quiet word like "*even pacing*" when hovered.
2. **One strip per scene.** Multi-scene aggregation is M6, not M5.
3. **One default metric shown.** The writer can toggle others on/off via a small affordance. Default = pacing. Active hue palette uses the existing semantic-glow tokens.
4. **Recompute on autosave only.** Same 2 s debounce as the pill engine. No live-mid-keystroke recompute.
5. **Drag to navigate.** Clicking a position on the strip scrolls the editor to the matching paragraph. (Replaces the "scroll forever to find that bit" friction.)
6. **Local-first.** Metrics are computed in Rust, persisted to SQLite, never round-trip the cloud.
7. **Degradation.** When LLM-based valence is unavailable (no provider configured), the strip shows only the locally-computed metrics (pacing, presence, world-refs). Coherence + valence fade out gracefully rather than disappear; their tracks render as faint placeholder fills with a "needs provider" hint on hover.

## 3. Where the Heatmap lives in the UI

```
┌─────────────────────────────────────────────────────────┐
│ [≡] Project name ▾                       saved · 12:48  │  scenes panel header
│  + new scene                             ┌─────────────┐ │
│  Scene 1 ◆                               │ ╭ Heatmap ╮ │ │  ← editor top strip,
│  Scene 2                                 │ ▒░▒▒░▓▒░░  │ │    24px tall
│  ✦ The Library                           │ pacing │ ▾  │ │
│                                          └─────────────┘ │
│                                                          │
│                                          THE LIBRARY'S   │
│                                          SECRET          │
│                                          ─                │
│                                          (body text...)  │
└─────────────────────────────────────────────────────────┘
```

**Location:** Inside the EditorCanvas, above the title hairline. Width matches `var(--water-canvas-max)` minus padding. Height 24 px. Right-edge "metric chip" shows the current metric name + a small ▾ that opens a popover of the five metrics; each row is a checkbox (Pacing / Valence / Coherence / Presence / World-refs) plus a one-line description.

Alternative considered: place the strip in the right margin alongside the pills. Rejected because (a) the margin is already pill territory, (b) the strip wants to span the scene's full width so position-along-the-strip maps 1:1 to position-in-the-text.

## 4. Interaction model

| Gesture                       | Effect |
|-------------------------------|--------|
| Hover a region of the strip   | Tooltip floats below the strip: *"end of paragraph 4 · even pacing"* — wording per metric. Cursor changes to indicate seekability. |
| Click a region                | Editor scrolls to the corresponding paragraph; the paragraph briefly flashes its hue-glow accent. |
| Drag along the strip          | Live-scroll the editor body (same as scrubbing through a video timeline). |
| Click the metric chip         | Popover opens; the writer toggles other metrics. Each toggled-on metric draws a separate semi-transparent layer over the strip, hue-coded. |
| Toggle a single metric off    | Track fades out, the strip rebalances. |
| Toggle ALL metrics off        | Strip collapses to a 6 px hairline (still draggable for navigation). |

**Discoverability:** the first time the writer opens any scene after M5 lands, a small one-time tooltip points at the strip with copy *"this is the heat. drag to find a spot."* and a dismiss-X. (Tooltip state persists in `localStorage` per the M4.5 polish pattern.)

## 5. The five metrics

For each metric: how it's computed, what hue it uses, what "high" and "low" feel like.

### 5.1 Pacing (default-on)

- **Computed:** rolling word-count delta per ~250 ms window over the scene's typing history (persisted across sessions via `scene_typing_history`, a new table). Fast bursts → high; long pauses + reflection → low.
- **Hue:** `--water-hue-pace` (warm vanilla).
- **Local-only.** No LLM needed.
- **Reads as:** density of fill. Fast-paced regions are dense vanilla; slow regions are nearly transparent.

### 5.2 Valence

- **Computed:** per-paragraph emotional tone (-1 cold / +1 warm). LLM call per paragraph on autosave; result cached in `heat_metric(scene_id, paragraph_ix, "valence", value, updated_at)`. Only paragraphs whose text-hash changed since last compute are re-evaluated.
- **Hue:** stacked. Positive → `--water-hue-valence-pos` (peach). Negative → `--water-hue-valence-neg` (icy-lavender). Mid → transparent.
- **LLM-required.** Falls back to placeholder when no router.
- **Reads as:** color swatches strung along the strip.

### 5.3 Coherence

- **Computed:** per-paragraph semantic-flow score vs. the previous N paragraphs. Embedding similarity (cosine) > threshold = coherent (high). Low cosine = sudden topic jump.
- **Hue:** `--water-hue-coherence` (periwinkle).
- **LLM-required.** Needs an embedding endpoint OR a small local embedding model (M5 ships with the LLM-API embedding path; local-model embedding is M6+).
- **Reads as:** continuous gradient — smooth = coherent, choppy = drift points.

### 5.4 Presence intensity

- **Computed:** count of characters referenced per paragraph (the M3 autosuggest scanner already does this). Normalized 0..1 by scene-max.
- **Hue:** `--water-hue-character-default` (or LRU among current characters' hues, faded).
- **Local-only.** Reuses M3 character-autosuggest scanner.
- **Reads as:** vertical bars — taller where more characters appear.

### 5.5 World-refs

- **Computed:** count of world-entry name+alias matches per paragraph. M4 `WorldRegistry::find_by_token` already provides this.
- **Hue:** `--water-hue-cartographer` (dune amber).
- **Local-only.** Reuses M4 world-drift Stage 1 scanner.
- **Reads as:** vertical bars at world-mention positions.

## 6. Engine architecture (sketch)

### 6.1 New SQL schema (v5)

```sql
-- M5: per-(scene, paragraph_ix, metric) rolling score cache.
CREATE TABLE heat_metric (
    scene_id      TEXT NOT NULL,
    paragraph_ix  INTEGER NOT NULL,
    metric        TEXT NOT NULL,
    value         REAL NOT NULL,
    text_hash     TEXT NOT NULL,
    updated_at    TEXT NOT NULL,
    PRIMARY KEY (scene_id, paragraph_ix, metric),
    FOREIGN KEY (scene_id) REFERENCES scene(id) ON DELETE CASCADE
);

-- M5: per-scene typing history. Append-only ring buffer (capped at a
-- few hundred entries) for the pacing computation. Older entries
-- migrate into per-paragraph aggregates and are then deleted.
CREATE TABLE scene_typing_history (
    scene_id    TEXT NOT NULL,
    ts_ms       INTEGER NOT NULL,
    word_delta  INTEGER NOT NULL,
    PRIMARY KEY (scene_id, ts_ms),
    FOREIGN KEY (scene_id) REFERENCES scene(id) ON DELETE CASCADE
);
```

Migration v5 lands in `crates/water-core/sql/v5_heatmap.sql`.

### 6.2 Rust surface

- `crates/water-core/src/heat/mod.rs` — top-level types (`HeatMetricKind`, `HeatRow`, etc.).
- `crates/water-core/src/heat/compute.rs` — `compute_pacing(history) -> Vec<f32>`, `compute_presence(paragraphs, characters) -> Vec<f32>`, `compute_world_refs(paragraphs, world_registry) -> Vec<f32>` — the three local-only metrics.
- `crates/water-core/src/heat/llm.rs` — `compute_valence(router, paragraph) -> f32`, `compute_coherence(router, paragraphs, ix) -> f32` — the two LLM-required metrics. Both are O(Δ): only paragraphs whose `text_hash` changed since the last cached row get re-evaluated.
- `crates/water-core/src/heat/store.rs` — `HeatStore::read(scene_id, kind) -> Vec<HeatRow>`, `HeatStore::write_batch(...)`, `HeatStore::invalidate(scene_id, paragraph_ix)`.

### 6.3 Trigger migration

`pace_floor` (M2 trigger) is currently a body-text heuristic. M5 replaces its evaluator with one that reads from `HeatStore`: when the trailing pacing window goes below a threshold, fire. Exit criterion: at least one trigger migrated and the existing pace_floor tests pass against the new evaluator.

`scene_flow_dip` (M2) similarly migrates to read from the coherence metric.

### 6.4 Renderer

- `app/src/heat/HeatmapStrip.tsx` — the 24 px-tall strip + drag-scrub handler.
- `app/src/heat/HeatmapMetricPicker.tsx` — the popover for toggling metrics.
- `app/src/heat/heatStore.ts` — subscribes to `heat:updated` events from the orchestrator, caches by scene id.
- `app/src/chrome/EditorCanvas.tsx` — mounts `<HeatmapStrip />` above the title hairline.

## 7. Exit criteria

Verbatim from spec § 4.7, plus M5-specific elaborations:

- [ ] Heatmap strip renders the default Pacing metric for the active scene within 500 ms of opening.
- [ ] Writer can toggle Valence/Coherence/Presence/World-refs and see each layer composite onto the strip.
- [ ] Drag along the strip scrolls the editor body within 1 paragraph of precision.
- [ ] Recompute completes within 1 s of an autosave landing on a 5 000-word scene.
- [ ] At least one M2 trigger (`pace_floor` or `scene_flow_dip`) reads its signal from `HeatStore` instead of the ad-hoc body heuristic.
- [ ] Heatmap degrades gracefully when no LLM router is configured: Pacing / Presence / World-refs render normally; Valence / Coherence render as faint placeholder fills with a hover hint.
- [ ] `cargo test -p water-core heat::` is green.
- [ ] Manual smoke walk against a new `eval/m5_acceptance/` fixture (1 scene with deliberate pacing/valence/coherence shapes) lands the expected strip pattern.

## 8. Explicitly NOT in M5

- Multi-scene heatmap aggregation (M6 candidate).
- Customizing the metric set per project (M7).
- Time-travel / playback of the heatmap evolution (M7+).
- Local embedding models for coherence (M6+, behind a feature flag).
- A11y / screen-reader narration of the heatmap (deferred to M7 polish).

## 9. Open questions — resolutions

The five open questions resolved at lock time:

1. **Strip placement** — Lock: editor-top, above the title hairline. Most discoverable; matches the writer's existing top-down reading order. If it ever feels intrusive after live use we revisit in M7 polish.
2. **Default metric** — Lock: Pacing on, all others off. Local-only, no LLM cost, low-noise. Writer opts into LLM-backed metrics explicitly via the metric picker.
3. **Tooltip wording** — Lock: deferred to a dedicated task in the plan (`heat/phrasebank.toml`). 8–12 phrases per metric, drafted carefully to read as observations rather than labels.
4. **Drag-to-scroll precision** — Lock: paragraph-level. Sentence-level adds enough text-position tracking complexity to push past M5 scope; deferred to M7 polish.
5. **LLM cost ceiling** — Lock: per-session token budget with a default of 20,000 input + 5,000 output tokens per session. Once exceeded, Valence/Coherence recompute pauses for the rest of the session with a quiet hint on the strip ("budget reached"). The budget is configurable in Settings → providers (M5 surface) and resets on project close.
